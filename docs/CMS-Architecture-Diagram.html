When you create/edit a product, you select the applicable attributes from each collection. The system automatically groups these by their linked ICON for display on the product card.

üéØ Key Architecture Points
  • ICONs are fixed visual elements - they never change per product, just display or hide based on what attributes are selected
  • ATTRIBUTEs are the variable data - these are what you select per product and what users filter by
  • Products reference ATTRIBUTEs, not ICONs - the system automatically determines which icons to show
  • Each ATTRIBUTE links to its parent ICON - this creates the grouping for display
  • One product can have multiple attributes - resulting in multiple icons displayed on the card
  • Filtering happens at the ATTRIBUTE level - users can filter by "Water-based" specifically, not just "Formula Base"
  • AI scraping populates ATTRIBUTEs - your agent selects from existing attribute options based on product analysis
  • Icons provide visual organization - they group related attributes under recognizable symbols with category colors
